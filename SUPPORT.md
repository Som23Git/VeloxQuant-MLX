# Support

## Documentation

Start with the official docs at
<https://veloxquant-mlx.netlify.app/docs/>. They cover installation, all
supported quantization algorithms, API reference, and integration guides for
`mlx_lm`.

## Reporting a Bug

Open an issue at <https://github.com/rajveer43/veloxquant-mlx/issues> and
include:

- your hardware (chip model + unified memory) and macOS version,
- `python`, `mlx`, and `mlx_lm` versions (`pip show mlx mlx-lm`),
- the model id you were running,
- a minimal code snippet that reproduces the problem, and
- the full traceback.

Without these details, the issue will likely be closed as unactionable.

## Asking a Question

For questions that are not bug reports — usage questions, design discussions,
ideas for new methods — open a GitHub Discussion or an issue tagged `question`.

## Feature Requests

Open an issue tagged `enhancement`. Describe the use case, not just the
proposed solution. If you have benchmark data supporting the request, include
it.

## Security Issues

Do not open a public issue for security vulnerabilities. See
[SECURITY.md](SECURITY.md) for the private reporting process.
