What is a KV cache?

Every time an LLM generates a token, it reads back the key and value tensors for every previous token in the context. These tensors are stored in RAM — that's the KV cache. At long contexts, it dominates memory: a 7B model at 128k tokens needs ~25 GB of KV cache alone, even on a compressed model.

What does VeloxQuant-MLX do?

It quantizes those tensors on the fly — trading a few bits of precision for dramatically less RAM. At 16× compression, a cache that needed 25 GB needs under 2 GB. Same model. More context. Runs on your Mac with near-identical output quality and unchanged mlx_lm API.

Where llama.cpp wins

It's the more mature, more portable project — runs on every platform, not just Apple Silicon. Its q4_0/q8_0 KV quantization is production-tested at massive scale through Ollama and LM Studio, and needs zero configuration. If you're not on a Mac, or you want the most battle-tested path with the least decision-making, that's the right default.

Where oMLX fits

A different axis entirely — it's a serving layer, not a compression scheme. Continuous batching, an OpenAI/Anthropic-compatible API, and SSD-tiered paging for long-idle sessions. Not mutually exclusive with VeloxQuant-MLX — both build on mlx_lm — but that combination isn't a supported path here today.

Where VeloxQuant-MLX wins

q4_0 is one fixed 4-bit scheme applied uniformly, with no eviction, ever. VeloxQuant-MLX goes lower — up to 16× key compression — with methods tuned for RoPE preservation, geometric key clusters, and per-layer budgets, plus 11 eviction methods that drop stale tokens entirely. All on top of the mlx_lm stack you're likely already running.

Algorithms

Forty-one compression strategies

Each method targets a different point on the compression–quality–throughput tradeoff curve. All are zero-calibration and mix-and-match per layer with RateQuant. Filter by trait below, or expand the release history.

What's new — recent releases
  • 0.39.0A2ATS-adapted — windowed RoPE + query-aware retrieval VQ (ACL 2025 Findings). Every VQ method already here applies RoPE uniformly (VecInfer: no RoPE-awareness at all; CommVQ-adapted: codebook-constrained, one exact rotation per position regardless of distance); A2ATS instead gates RoPE precision by each token's distance from the current decode position — exact within a trailing window, a single shared fixed-offset approximation outside it — combined with query-aware codebook assignment for a retrieval-fraction subset of tokens. This is a normal-track method: a live-verified peer-reviewed venue, no exception needed. The honest crux: no query is visible at cache level (the incoming key is used as a proxy query, same category as H2O/SnapKV/AMC); requires an offline codebook calibration pass, not auto-invoked. Measured, reported plainly rather than softened: windowed RoPE is worse than always-exact RoPE in every geometry tested — ~2.8× higher reconstruction MSE on a positional-locality-favorable geometry, ~4.4× higher on a long-range-dependent one; query-aware VQ assignment has higher reconstruction MSE than plain nearest-centroid VQ in every row measured (mathematically expected — the intended downstream retrieval-quality payoff isn't something this benchmark can show).
  • 0.38.0AMC-adapted — saliency-driven tiered rank + precision (no verified peer-reviewed venue — a second one-time, stated exception; the paper's hardware/RTL half is entirely out of scope). Every rank-adaptive method already here (Palu) and every bit-width-adaptive method (KIVI, SKVQ, RateQuant) adapts one axis; AMC is the first to drive both rank and bit-width from a single per-token L1-norm saliency score, via three tiers (High: rank 128/16-bit, Mid: rank 43/8-bit, Low: rank 8/4-bit). The honest crux: compression-only, never eviction (every token retained, unlike every eviction method here); query-aware saliency and closed-loop adaptive thresholds are opt-in, off by default; requires an offline SVD/PCA calibration pass, not auto-invoked. Measured: on a planted sparse-outlier geometry, AMC beats a matched-byte-budget uniform baseline by ~8× lower reconstruction MSE; honestly, on a uniform-magnitude (no-signal) geometry AMC is ~100× worse than the same baseline — reported plainly. The paper's own 59.2%-energy/2.24×-throughput/3.6%-accuracy numbers are hardware-measured on the paper's own 45nm RTL simulation — not reproduced.
  • 0.37.0NestedKV-adapted — multi-scale ensembled prefill eviction (no verified peer-reviewed venue — a one-time, stated exception). Every eviction method already here (H2O, SnapKV, CurDKV, PyramidKV, Keyformer, MorphKV, KVzip, ...) scores a token from one importance signal; NestedKV keeps three parallel key-only continuum-memory statistics — stable/global, episodic/block-local, current/recent-window — and combines them via a training-free head-adaptive blend and a per-token surprise-gated route. The honest crux: a one-shot prefill compressor (unlike every other eviction method here, the cache is not bounded during decode — new tokens are simply appended, never rescored), and all gate/blend constants (beta=3.0, tau=0.60, kappa=10.0, safeguard_alpha=0.20) are taken directly from the paper's own Appendix A. Measured: on planted single-scale anomaly geometries, NestedKV retains the anomaly in 100% of trials vs H2O's 0% for both the global and recency cases; on the local-episodic case, honestly, neither method protects it at this benchmark's synthetic scale — a structural finding about how the blend/gate constants interact at small N, not a bug. The paper's own RULER/LongBench/LooGLE/InfiniteBench/MMLU-Pro numbers are the paper's, on trained models — not reproduced.
  • 0.36.0CurDKV-adapted — value-aware leverage-score eviction via approximated CUR decomposition (NeurIPS 2025). Every eviction method already here (H2O, SnapKV, TOVA, PyramidKV, Keyformer, MorphKV, KVzip, ...) scores a token using only its key side; CurDKV derives a leverage score from the joint key-and-value structure of the proxy attention output, so a token's value contribution — not just its key/attention profile — decides whether it survives. The honest crux: key-as-query proxy (same as H2O/SnapKV), an SVD-based energy-weighted leverage-score estimator (not the paper's own CUR sampling routine — a hard top-k cutoff degenerates to uniform leverage whenever the retained rank reaches the block size), and newly-appended tokens are seeded with their own leverage score rather than a flat 0. Measured: on a planted geometry (near-identical keys, sharply divergent values), CurDKV retains value-relevant tokens preferentially in 8/8 trials across seeds, while H2O — given the identical keys — cannot tell the classes apart. Two tokens with identical keys but different values provably receive different CurDKV scores. The paper's up-to-9.6%-higher-accuracy and up-to-40%-latency-reduction numbers are the paper's, on trained models — not reproduced.
  • 0.35.0KVTC-adapted — local PCA + DP-optimal per-component bit allocation + entropy coding (ICLR 2026). Palu/SVDq/SpectralQuant all use a fixed mixed-bit split (a hand-chosen top-25%/75% tier, or a binary signal/noise cutoff); KVTC computes a dynamic-programming-optimal integer bit-width per principal component under a hard total-bit budget — including exactly 0 bits (dropping a component entirely) — then entropy-codes the resulting codes. The honest crux: local per-sequence PCA (not the paper's pre-calibrated global basis), an analytic Gaussian distortion proxy reused from ratequant.py (not a real-activation-fit rate-distortion model), and a plain order-0 Huffman coder (not the paper's possibly more sophisticated scheme, never the Shannon-entropy bound). Measured: at a matched total byte budget on planted skewed-variance geometry, the DP allocator reaches mean MSE ≈0.027 vs ≈87.6 (fixed-uniform) and ≈84.4 (SVDq-fixed-split); on a flat control it is roughly competitive with the uniform baseline, not a dramatic win. The paper's up-to-20×/40×, under-1pp-accuracy-loss numbers are the paper's, on trained models — not reproduced.
  • 0.34.0KVzip-adapted — context-reconstruction reliance retention (NeurIPS 2025 Oral). Every other proxy scorer ranks a stored token by the attention it receives from a query (cumulative for H2O, latest for TOVA, windowed for MorphKV); KVzip ranks by reconstruction reliance — how much the model relies on a KV pair to reconstruct its own context — a query-agnostic profile computed once and reused across all future queries. The honest crux: key-as-reconstruction-probe proxy (a cache never runs the model to reconstruct text), and only the kvzip_probe="latest" reduction is pinned — it is TOVA-adapted, bit-for-bit; no H2O collapse is claimed. Measured: under a constructed reconstruction shift, cumulative H2O retains ~0.017 of the reconstruction-critical region while the context probe retains ~0.609, beating the probe=latest (TOVA) reference (~0.248); a flat control shows no advantage. The paper's accuracy/memory numbers are the paper's, on trained models — not reproduced.
  • 0.33.0MorphKV-adapted — recent-window correlation retention (ICML 2025). Where H2O ranks a stored token by cumulative attention (inertial — early heavy hitters crowd out the current topic) and TOVA by the single latest query, MorphKV ranks by correlation with a sliding window of the most recent tokens, so a constant-size cache re-targets toward what the recent context actually reads — eliminating the "early-token bias." The honest crux: key-as-query proxy (a cache never sees the true query), and only the morphkv_window=1 reduction is pinned — it is TOVA-adapted, bit-for-bit; no H2O collapse is claimed. Measured: under a constructed topic shift, cumulative H2O retains ~0% of the recent-relevant region while wider windows retain ~0.39–0.74, beating the window=1 (TOVA) reference; a stable control shows no advantage. The paper's accuracy/memory numbers are the paper's, on trained models — not reproduced.
  • 0.32.0Keyformer-adapted — Gumbel-regularized heavy-hitter eviction (MLSys 2024). Structurally H2O's proxy-attention accumulator plus one new ingredient: Gumbel noise on the eviction logits, so a "late riser" — a token that reads low early, before the queries that attend to it arrive — is not deterministically pruned before it can recover. The honest crux: the paper redraws and anneals the noise across generation, but a cache has no trustworthy global step, so we draw one deterministic per-position Gumbel value and freeze it (keyformer_seed) — the intent, not the schedule. Setting keyformer_tau=0 removes the noise and this cache is H2O-adapted, bit-for-bit — the honest ablation. Measured: under late-riser geometry greedy tau=0 evicts the planted riser 100% of the time while tau=6 rescues it ~75%; the downstream perturbation is a noisier, regime-dependent secondary effect, reported as-is.
  • 0.31.0Q-Filters-adapted — query-agnostic projection eviction (preprint), the library's fourth eviction scorer class. Score each key by its projection onto a frozen per-head direction. The honest crux: the paper derives that direction from query SVD offline, but a cache never sees queries, so we substitute the SVD of the first observed keys — which recovers the dominant axis but not which end is important, making qfilters_sign a genuine ablation and the kept set path-dependent. Measured: key-SVD recovers the planted axis at filter_cosine ≈ 0.97; the correct-sign arm beats random by +0.16 mean perturbation under paper-like geometry, ~nothing under the isotropic control — both reported.
  • 0.30.0SKVQ-adapted — sliding-window quantization with two mechanisms new to the library (COLM 2024): channel reordering (per-head permutations sort like-range channels into the same quant group, frozen from the first flushed chunk — no calibration) and clipped dynamic quantization (per-group grid-searched clip factor; α=1 always in the grid, so the search never loses under its own metric). Both K and V behind a sliding fp16 window + sink filter; bit-for-bit path-independent. Measured: reordering −16.9% key MSE on top of clip search under channel heterogeneity, −0.3% (nothing) under the homogeneous control — both reported.
  • 0.29.0L2Norm-adapted — first intrinsic-signal eviction cache. Token importance is read directly off the stored key's L2 norm (EMNLP 2024 finding: low norm ⇒ high attention in trained LMs) — no attention scores, no key-as-query proxy. Zero per-step scoring cost (~100–800× faster than H2O at prefill on the committed harness) and bit-for-bit path-independent kept sets: prefill and token-by-token decode yield identical caches.
  • 0.28.0NSNQuant-adapted — first calibration-free distribution-matching VQ. Instead of fitting a codebook to the data, a Normalize-Shift-Normalize transform + Hadamard rotation reshapes K/V tokens onto the standard normal, so one universal codebook built offline from synthetic Gaussian samples quantizes any model at 1–2 bits/element. Both keys and values; chunk-flush fp16 residual buffer; +0.04–0.11 reconstruction cosine over the same VQ without NSN on channel-biased inputs, collapsing to ~0 when input is already centered.
  • 0.27.0xKV-adapted — third cross-layer method. A fixed-size group of layers jointly factorizes their stacked key matrices into one shared SVD basis via a fan-in/fan-out coordinator, amortizing the basis storage cost across every member. 8–20% fewer bytes than independent per-layer SVD at matched reconstruction quality, improving with group size.
  • 0.26.0CaM-adapted — first cache-merging cache. Makes the same eviction choice as H2O but merges the evicted token into the survivor it most resembles (cosine-weighted) instead of dropping it, recovering a share of the lost mass at high compression. No coordinator; cam_merge=drop reduces bit-for-bit to H2O.
  • 0.25.0ChunkKV-adapted — first chunk-level eviction cache. Partitions the sequence into contiguous chunks and keeps or drops each chunk as a whole by pooled importance, preserving local coherence that token-level eviction shreds. No coordinator; chunk_size=1 reduces bit-for-bit to H2O.
  • 0.24.0SqueezeAttention-adapted — first 2D (layer × token) data-driven budget cache. Measures each layer's attention concentration at prefill and reallocates a fixed total budget toward broad layers via a one-shot coordinator re-budget. The data-driven sibling of PyramidKV; strength=0 == H2O.
  • 0.23.0PyramidKV-adapted — first layer-adaptive budget cache. H2O-style eviction with a pyramid of per-layer budgets (large early, small deep, fixed average), funneling memory to where attention spreads. Flat pyramid == H2O.
  • 0.22.0TOVA-adapted — first memoryless eviction cache. Scores tokens by the attention weight at the current step only (no accumulation), evicting the token no longer attended to. The reactive counterpart to H2O.
  • 0.21.0H2O-adapted — evicts by cumulative attention mass. A small set of "heavy hitter" tokens receive most attention across the sequence; scoring by running mass keeps them at constant memory.
  • 0.20.0StreamingLLM-adapted — first constant-memory cache. Keeps the first stream_n_sink sink tokens and the most recent stream_window_size tokens; the cache never grows beyond a fixed bound.
  • 0.19.0SnapKV-adapted — first token eviction method. Retains only the top-snap_budget tokens from prefill by observation-window attention scoring, dropping the rest.
  • 0.18.0ZipCache-adapted — first saliency-adaptive cache. A key-norm proxy ranks tokens, routing a small top fraction to hi_bits while the rest stay at lo_bits.
  • 0.17.0GEAR — first error-feedback cache. Reconstructs the quantization error with a low-rank residual plus sparse outlier correction (X ≈ Quant_b(X) + L·R + S).
  • 0.16.0CacheGen — first entropy-coded cache (delta-codes then compresses toward Shannon entropy). MiniCache merges adjacent middle-to-deep layers' KV into one shared SLERP direction.
  • 0.15.0PALU — first true low-rank latent cache. Stores keys and values as latent [S, r] codes instead of full fp16, for a full-KV rate below 1 bit/element.

CaM-adapted
cache merging — merge, don't drop
Merges each evicted token into the survivor it most resembles instead of dropping it.
merge vs drop · cam_merge=drop == H2O · 32/32 tests
Details
Adapted from CaM (Zhang et al., ICML 2024, PMLR 235). The library's eighth eviction configuration and the first on the merge-vs-drop axis. Every other eviction method permanently drops the tokens it evicts; CaM instead merges each evicted token into the surviving token it most resembles (a cosine-weighted blend of the value rows, and optionally the keys), then removes only the redundant slot — so the information is folded into a neighbour rather than lost. Set cam_merge="drop" and it reduces bit-for-bit to H2O.

① Choose — same loser H2O would evict (lowest cumulative attention mass, sinks protected)
② Target — the surviving non-sink token whose key is most cosine-similar to the loser
③ Merge — x_new = (1-w)·x_survivor + w·x_evicted, w = clip(cos, 0, 1); survivor inherits the mass, loser's slot removed

Why merge: eviction always perturbs the output — the dropped token still carried mass. Folding it into a neighbour recovers a share of the lost signal exactly where dropping hurts most. Measured perturbation drops from 0.955 → 0.708 at 16× vs plain H2O.

Details
Adapted from Devoto, Zhao, Scardapane & Minervini (EMNLP 2024, arXiv:2406.11430). The library's ninth eviction configuration and the first with an intrinsic scorer: every other eviction method scores with attention / a key-as-query proxy or pure position; L2Norm reads importance off the stored key itself. The paper's counterintuitive trained-LM finding: keys with low L2 norm attract high attention — so keep the lowest-norm tokens. (Note the sign inversion vs ChunkKV's key_norm option.)

① Norm once — each incoming key's L2 norm is computed at insertion and never updated
② Top-k — over budget, one protected top-k keeps the lowest-norm tokens (sinks always survive), in original order
③ No loop — no per-token softmax-over-cache: 0.3–1.2 ms per prefill block vs H2O's 37–275 ms on the same harness

Why it's different: because the score is intrinsic, the kept set is path-independent — prefill-in-one-block and token-by-token decode produce bit-for-bit identical caches (pinned by test; no accumulating-score method has this). Honest scope: the low-norm ⇒ high-attention correlation is the paper's empirical claim about trained models — under an isotropic synthetic control the advantage reverses, reported in full in the committed benchmark.

Q-Filters-adapted

Scores each key by its projection onto a frozen per-head direction — attention-quality importance with no attention, no proxy, no per-step cost.
query-agnostic projection · no attention, no proxy · 27/27 tests
Details
Adapted from "Q-Filters: Leveraging QK Geometry for Efficient KV Cache Compression" (arXiv:2503.02812, preprint). The library's fourth eviction scorer class — after attention/proxy, structural, and intrinsic-norm. The paper's premise: trained heads have an anisotropic QK geometry, so a single per-head direction predicts attention.

① Calibrate — freeze a per-head filter as the top singular vector of the first observed keys
② Project — score every key by sign · (key · filter), computed once at insertion
③ Top-k — over budget, keep the highest-scoring tokens (sinks always survive), in original order

The honest crux: the paper derives the filter from query SVD offline, but a cache never sees queries — so we substitute the SVD of observed keys. That recovers the dominant axis but not which end is important (the sign a query would fix), making qfilters_sign a genuine ablation and the kept set path-dependent. Measured: key-SVD recovers the planted axis at filter_cosine ≈ 0.97; correct-sign beats random by +0.16 under paper-like geometry, ~nothing under the isotropic control — both reported. uint vec_idx = thread_position_in_grid.x;
uint N_total = x_shape[0];
if (vec_idx >= N_total) { return; }

uint n_centroids = codebook_shape[0];
uint sub_dim     = codebook_shape[1];
uint x_base      = vec_idx * sub_dim;

float best_dist = INFINITY;
uint  best_idx  = 0;

for (uint c = 0; c < n_centroids; ++c) {
    uint cb_base = c * sub_dim;
    float dist = 0.0f;
    for (uint i = 0; i < sub_dim; ++i) {
        float d = float(x[x_base + i]) - float(codebook[cb_base + i]);
