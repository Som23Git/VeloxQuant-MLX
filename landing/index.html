Algorithms

Four quantization strategies

Each method targets a different point on the compression–quality tradeoff curve.

RVQ
TurboQuant RVQ
Two-pass scalar residual vector quantization. 1-bit, 2-bit, and 3-bit codebooks fitted to Gaussian + Laplacian key distributions. Achieves up to 7.5× compression with tight distortion bounds.
NEW
RateQuant
Per-layer bit allocation via reverse-waterfilling (arxiv:2605.06675). Automatically assigns more bits to high-sensitivity transformer layers. Delivers equal or better quality at lower average bit-width.
POLAR
PolarQuant
Recursive polar coordinate decomposition for clustered key distributions. Lossless angular encoding preserves inner-product structure critical to attention score accuracy.
1-BIT
QJL
Johnson-Lindenstrauss 1-bit sign sketch. An inner-product-preserving random projection for extreme compression scenarios. Minimal memory footprint with provable approximation guarantees.

Quickstart

Up and running in 5 lines

Import, configure, build — then pass the cache directly to mlx_lm.

python 
from veloxquant_mlx import KVCacheBuilder, KVCacheConfig

# Build a TurboQuant RVQ cache at 1-bit
cfg = KVCacheConfig(
    method="turboquant_rvq",
    bit_width_inlier=1,
    seed=0,
)
cache = (
    KVCacheBuilder()
    .with_method("turboquant_rvq")
    .with_head_dim(128)
    .with_bit_width(1)
    .with_seed(0)
    .build()
)
python — RateQuant per-layer allocation 
from veloxquant_mlx import allocate_bits_ratequant, calibrate_layer_sensitivities

# 1. Probe the model to get per-layer sensitivity scores
sensitivities = calibrate_layer_sensitivities(model)

# 2. Allocate bits — average 1.5 bits across all layers
bits = allocate_bits_ratequant(sensitivities, target_avg_bits=1.5)
# → [2, 1, 1, 3, 1, 2, ...]  one int per attention layer

# 3. Build per-layer caches and inject into mlx_lm
caches = KVCacheBuilder.for_model(model, KVCacheConfig(
    method="turboquant_rvq", bit_width_inlier=bits, seed=0,
))
